# 从转录蒸馏新启发式的标准流程

## 触发条件
用户说"蒸馏"、"用今天的视频更新skill"、"看看有没有新启发式"等。

## Step 0: 判断输入来源

蒸馏的输入有两种来源，处理流程不同：

| 来源 | 特征 | 处理方式 |
|------|------|---------|
| **转录文件** | 用户说"今天的视频"、提供video ID | → Step 1（定位转录文件） |
| **手动文本** | 用户直接粘贴发言内容（点位+分析） | → 跳过Step 1，直接从Step 2开始，用用户提供的文本作为输入 |

**手动文本蒸馏要点**：
- 用户粘贴的内容已经是精华（点位+原话+分析），不需要再从转录中提取
- 直接进入Step 3（与现有skill对比），找出增量信息
- 点位数据直接记录，原话引用直接使用用户提供的原文
- 案例：5/22晚间发言蒸馏——用户粘贴早晚两段发言，直接对比找出3个增量（高密度争夺区、反弹失败回落、分界线大幅下移）

## Step 1: 定位转录文件

```bash
# 查找最新转录
ls -lt /mnt/h/大镖客蒸馏/*/
```

文件命名规则：
- `{video_id}_fw.txt` — faster-whisper转录（主力）
- `{video_id}_transcript.txt` — 字幕API转录（备用）
- 优先使用 `_fw.txt`（准确度更高）

## Step 2: 读取转录内容

按200行分段读取，重点关注：
1. **关键点位**：核心关键位、多空分界线、压力支撑位
2. **指标分析**：RSI/MACD/KDJ/DMI/ATR/布林带/维加斯通道的跨周期状态
3. **持仓量分析**：OI变化+价格方向的组合判断
4. **操作建议**：大镖客给出的具体操作逻辑
5. **原话引用**：大镖客的原话（用于启发式案例）

## Step 3: 与现有skill对比

```bash
# 搜索现有启发式
grep "启发式" ~/.hermes/skills/dbk9527-perspective/SKILL.md | head -70
```

对比逻辑：
- 已覆盖的启发式 → 跳过或强化（补充新案例）
- 部分覆盖 → 强化现有启发式（添加新维度）
- 完全新增 → 新增启发式

## Step 4: 更新skill

### 4.0 更新清单（必须全部完成，漏一项=校验失败）

每次蒸馏必须更新以下所有位置，按顺序执行：

1. **frontmatter** — `heuristics_count` + `video_count` + `last_updated` + `latest_verified`
2. **description** — "合计X条" + "39-X号" 两处数字
3. **新增/强化启发式** — 正文
4. **当日关键点位** — 新增section
5. **今日技术面总结** — 更新或新增
6. **诚实边界** — "信息截止到..."
7. **启发式编号说明** — "共X条（38条基础+Y条新增）"

**⚠️ 用户对数字准确性要求极高**：曾因frontmatter写82条实际只有24条被发现并要求修复。更新后必须立即自检：frontmatter的heuristics_count是否=description中的"合计X条"=实际启发式总数=编号说明中的"共X条"。

### 4.1 更新frontmatter
```yaml
heuristics_count: X+1  # 新增数量
video_count: Y+1       # 新视频数量
last_updated: YYYY-MM-DD
latest_verified: YYYY-MM-DD（简要描述当日分析核心）
```

### 4.2 更新description
- "合计X条" → "合计X+N条"
- "39-X号" → "39-X+N号"
- "#XXXX" → "#XXXX+1"

### 4.3 新增启发式（如需要）
格式：
```markdown
### 新增启发式N：标题
- **应用场景**：...
- **大镖客原话（日期）**：「原文引用」
- **关键推论**：...
- **操作含义**：...
- **与其他启发式关系**：强化/补充/新增
```

### 4.4 强化现有启发式（如需要）
在原启发式末尾添加：
```markdown
- **补充（日期强化）**：新内容
  - 大镖客原话（日期）：「原文引用」
```

### 4.5 添加当日技术面总结
```markdown
## YYYY-MM-DD 蒸馏新增（基于日期转录 #编号）

### 关键点位（日期）
- 核心关键位: $XX,XXX
- 日线多空分界线: $XX,XXX
- 压力位: ...
- 支撑位: ...

### 新增/强化内容...

### 今日技术面总结
- 格局判断
- 关键信号
- 时间窗口
```

### 4.6 更新诚实边界
- "信息截止到..." → 更新为最新视频编号

### 4.7 更新启发式编号说明
- "共X条（38条基础+Y条新增）" → 更新数量

## Step 5: 校验一致性

```bash
cd ~/.hermes/skills && git add dbk9527-perspective/SKILL.md && git commit -m "..."
```

提交时会自动运行校验脚本，检查：
- frontmatter的heuristics_count与实际启发式数量是否一致
- 编号是否连续无断号

## Step 6: 提交git

```bash
git commit -m "更新大镖客skill：新增启发式X-Y + 强化启发式Z

基于日期转录（#编号）蒸馏：

新增启发式X：标题
- 关键内容

强化启发式Z：补充内容
- 关键内容

更新内容：
- heuristics_count: A→B
- video_count: C→D"
```

## 常见陷阱

1. **frontmatter数字必须与实际内容一致** — 用户对此要求极高，曾因82条vs24条不一致被发现并要求修复。更新后必须立即自检所有数字（见4.0清单）
2. **不要遗漏description中的数字更新** — description中的"合计X条"和"39-X号"也必须同步更新
3. **启发式编号必须连续** — 不能跳号，不能重复
4. **大镖客原话必须准确引用** — 不能改写，必须用原文
5. **强化现有启发式时，要标注"补充（日期强化）"** — 区分原始内容和新增内容
6. **避免重复内容** — 当在原启发式末尾添加"补充（日期强化）"时，不要再创建独立的"强化启发式N"章节。2026-05-22教训：新增启发式48已有补充内容，又创建了独立的"强化启发式48"章节，导致内容重复。修复方案：删除独立章节，保留原启发式中的补充内容
7. **手动文本蒸馏也要走完整更新清单** — 用户直接粘贴发言时，容易因为"只是补充"而跳过部分更新位置（如description、编号说明）。5/22教训：即使只是新增2条启发式+强化1条，frontmatter/description/编号说明/诚实边界也必须全部更新
8. **强化启发式时，补充内容放在原启发式末尾，不要改原启发式的标题或结构** — 只添加"补充（日期强化）"段落，保持原有内容不变
